Crowley and the Cheese Covenant

Crowley and the Cheese Covenant is an advernture platforming game where you help navigate Crowley out of a dungeon while collecting cheese to power the teleporters and also avoiding deadly traps! This game features animations, a story, cutscenes, 8 levels, and takes around 30 minutes to beat. Link to a downloadable version. Link to the github repository. Link to itch.io. The downloadable version runs the best by the way.

Technical details

Game objects, levels, and cutscenes were all handled via C# scripts. Linear traps, the ones that only move in one dimension, were created by taking advantage of either cosine or sine waves in conjunction with linear interpolation. I first determined the end position coordinates for each trap and then had a variable that served as the x and y coordinates of the trap. This variable would constantly be updating the position of the trap by matching it's value to the constantly changing trigonometric waves. This variable would interpolate between these two end points by "riding" the trigonometric waves since these waves are always fluxuating between -1 and 1. These traps can be moved in reverse by simply multiply the result of these waves by negative one before setting the positioning variable to the wave. A similar effect was used to create the circular traps however these traps did not use linear interpolation but just trigonometric waves but this time the variable used the trigonometric waves for both its x and y coordinates rather than just one like the linear traps. Both the levels system and dialogue system were implemented by storing information in an array that was constantly being updated and parsed to retrieve the correct information to smoothly move from level to level and dialogue to dialogue.

Sudoku Solver

This program takes a sudoku problem in the form of a text file and solves it using backtracking and recursion in a tree structure. Complete with JUnit testing and a few example problems. This algorithm can solve a sudoku problem with any dimensions as long as it is a perfect square (even a 400x400 board! That is if you're willing to wait a while). This program works by reading text files like this where the first number indicates the dimensions of the puzzle and the dots represent empty spaces. You can find the code here.

Technical details

A internal 2D array is instaniated at runtime to create a platform for the algorithm to attempt to solve the puzzle. If the algorithm managed to get to the last index of the 2D array, then the puzzle is deemed solvable and fed to another algorithm that prints out the first solution obtained. The algorithm works by essentially turning every possible solution into a tree and after every attempted insertion, it checks if that was a legal insertion into the puzzle. If not, the algorithm moves onto another branch of the tree to try another possible insertion. If no insertions are possible at this level, the algorithm moves up the tree by going to the previous index of te 2D array and tries a different solution. This process is repeated until all tree branches and roots are exhausted, or the last index is filled. In the case of the former, this is when the algorithm determines that the puzzle is unsolvable.
